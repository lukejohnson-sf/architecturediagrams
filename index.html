04

SLAS Authentication Flow

Shopper login and API access service token lifecycle

🌐
Browser
1
Guest Token Request
Browser requests guest PKCE token from SLAS
→ SLAS Public Client
2
Access Token Issued
JWT access + refresh tokens returned, stored in httpOnly cookie
← Access Token (JWT)
3
API Calls via SDK
Commerce SDK attaches bearer token to all SCAPI requests
→ SCAPI Zone
4
Login (Registered Shopper)
PKCE authorization code flow — shopper logs in, code exchanged for tokens
↔ SLAS / IdP
5
Token Refresh
Refresh token silently rotates access token before expiry
→ SLAS Token Endpoint
